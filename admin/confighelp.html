Transceiver Settings

 

The Transceiver Settings menu selection is where the Developer and/or Administrator will configure the Transceiver Transceiver.  Setting the Transceiver’s configuration is a simple process and all updates are easily applied via the Save button on the Transceiver Settings page.  However, new settings will not take effect until the Transceiver Transceiver is restarted.

 

The following list describes each of the configurable settings and the impact they have on the Transceiver:

 

Transceiver Classpath – This setting augments the system classpath.  CAUTION: this setting is global to the Transceiver, individual handlers should set their classpath’s in the handler registry.

 

Registry File – Location of the registry file.  Usually this setting is not changed, however, it becomes useful in a multi-Transceiver (Transceiver farm) environment.

 

Transceiver TCP Port – The TCP port on which the Transceiver will listen for HTTP connections.  The system default for this setting is 8082.

 

Admin TCP Port – The TCP port listening for Transceiver web-based administration (HTTP).  The default setting for this field is 44411.

 

Request Timeout – From this field you can determine how long the actual Socket connection will remain open before timing out.  The default setting for this field is 120 (seconds).

 

Socket Listeners – This setting limits the number of HTTP connections that can connect at one time.  The default setting for this field is 50.

 

Handler Thread Pool – This setting is used to limit the number of Asynchronous and Synchronous requests that can be executing at one time.  The default setting for this field is 30.

 

SMTP Host – This field is a simple mail client used to forward asynchronous responses, error handling, or fault tolerance to an SMTP Transceiver (ie; Exchange Transceiver).  All requests must be sent to a resolvable Transceiver.

 

Verbose Logging – This field is used to determine whether or not to use Verbose Logging.  The choices that can be made are Yes to turn on Verbose Logging or No to turn it off.  The system will default to No.  This setting enables logging within the Java console.

 

Logging Path – This setting is used to determine what directory to store log files in.  The folder must be created prior to setting the path.

 

Web Serving Path – Use this field to set the directory where static HTML pages and images are stored.

 

Servlet Directory – This setting is used to direct the Servlet to either a specific classpath JAR file or folder (depending how you use the system).

 

Servlet Auto Reloading – Auto Reloading is used primarily in a development environment to automatically check classes for modification and reload Servlets that have been re-compiled without having to restart the Transceiver.  This field enables that feature.  The default is Yes.  However, Class Auto Loading should be turned off in a production environment because it slows the Transceiver.

 

XSL Path – This setting is used to direct handlers to where XSL (XML Style Language) are stored.  Handlers will seek style sheets based on the specifications in this field.

 

Use SSL – This field is used to determine whether or not you use SSL (Secure Socket Layer) for HTTP transactions.  You can select Yes to use SSL or No to decline.  The default setting for this field is No.  Using SSL requires a Certificate Authority License (ie; Verisign).

                                                                                                                       

SSL TCP Port – Use this setting to determine the port number that the Secure Socket Transaction will accept.  The system default is automatically set to 443 upon enabling SSL.  However, the system default will be set to 0 if SSL is not enabled.

 

Java Key Store for SSL – This field is the Java Key Store (JKS) file where the SSL certificates are stored.

                                                                                                                       

Passphrase to JKS for SSL – The passphrase for the Java Key Store.

 

Use Fault Tolerance – This field is used to turn on fault tolerance.

 

Fault Tolerance Database Driver – This field is used to define the appropriate driver for the database that you are building fault tolerance in.

 

Fault Tolerance Database URL – The database driver is entered into this field in order to determine the database that is being used.

 

Fault Tolerance Pooled Connections – The number of pooled database connections used for fault tolerance.

 

·         Note: The Save button must be clicked once you have either set or changed any of the Transceiver settings.  However, settings will not take effect until the Transceiver is restarted.  The Transceiver can be shutdown from the main administration screen.  Failing to specify correct paths to configuration files can result in failure of the Transceiver to start or operate correctly.  Exercise caution here.
