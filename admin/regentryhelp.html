XPATH Identifier – Name of the element tag that identifies the transaction.

 

Class – Name of the Java class that is the Plugin.  This class must be fully qualified with package name.

 

Description – A simple (text) description of the Plugin.

 

Plugin Type – This dropdown menu is used to determine the type of Plugin.  There are four Plugin types to choose from (Object, EJB, Event, Document, Scripted).  The system defaults to Object.

 

Plugin Classpath – Define the CLASSPATH specific to this Plugin.  Usually set to the location of the repository jar compiled by the developer console.

 

Response Type – This dropdown is used to determine the response type.  The two specified types are Sync and Async.  The system defaults to a response type of Synchronous.  Synchronous response types wait for a reply from the recipient before fully completing.  Asynchronous response types will send the transaction and move on to the next transaction before receiving a response from the recipient.

 

Respond to URL – This field is only used (and valid) when Asynchronous is selected as the Response Type.  It is used to determine where to send the return response.

 

Retries before Failure – This field is used to define the number of times the transaction will retry before terminating.

 

Plugin Timeout – This field can be used to set a limit on a Plugin before it times out.

 

Transaction Priority – This field enables the Developer or Administrator to increase the Plugin’s thread priority in order to allow the thread more time to process.  This setting is extremely useful for threads that need to process immediately.  The default is ‘Unspecified’ and should be sufficient in most cases.

 

Plugin Script File – This field is where the name of the script file run by the Plugin needs to be entered.

 

XSL Style Sheet – A style sheet can be identified in order to transform styles that need to be applied to return results.

Application Name – This field must be a unique field that identifies the Fault Tolerance database tables associated with this Plugin.  This value is should be retrieved from the generated SQL (it is the prefix to the table names) from the developer console project directory.

 

None – No action is taken.

 

Track Only – Failed transactions are logged in the Transceiver log file.

 

Notify – A notification is sent to a URL that has been designated in the Error Respond to URL field.

 

Auto Restart – Failed transactions will be resubmitted when the database is restarted.  In addition, an e-mail message will be sent.

 

Error Respond to URL – This field is used in conjunction with the Fault Tolerance Action field.  The Developer or Admin can specify an e-mail address to send a fault tolerant condition to by using the mailto: command.

 

Mail From Address – This field is used to specify the mail from address when sending a fault tolerant action.
